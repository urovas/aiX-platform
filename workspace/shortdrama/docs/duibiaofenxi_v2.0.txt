对标分析
——行业概览与顶尖博主模块 - 软件需求规格说明书 v2.0
1. 引言
1.1 编写目的
本文档旨在明确“短视频博主对标分析系统”中【行业概览与顶尖博主】模块的功能需求、数据规范和技术约束，为系统设计、开发和测试提供依据。
1.2 适用范围
本模块面向所有用户类型（新手创作者、资深创作者、MCN机构），提供短视频/短剧行业的宏观数据、平台格局、内容领域分类及顶尖博主核心指标。
1.3 术语表
术语	定义
对标分析	指对特定博主的深度数据拆解和特征提取
拆片分析	指对单个视频作品的逐帧分析和特征提取
互动率	(点赞数+评论数+分享数)/播放量 ×100%
Qwen-VL	本地部署的多模态大模型，用于视觉内容分析
1.4 参考文献
•	《中国网络视听发展研究报告（2025）》
•	DataEye短剧数据接口文档 v2.0
•	新榜开放平台API文档 v3.0
2. 总体描述
2.1 产品视角
本模块作为对标分析系统的宏观数据入口，通过可视化仪表盘和分类导航，帮助用户快速建立对行业格局的认知，定位可对标的头部玩家。
2.2 用户特征
用户类型	使用频次	核心诉求
新手创作者	每周1-2次	赛道选择、寻找对标对象
资深创作者	每周3-5次	竞品监测、趋势跟踪
MCN机构	每日使用	签约评估、行业报告生成
2.3 业务规则
1.	所有榜单数据每日凌晨2:00自动更新
2.	博主核心指标基于近30天视频数据计算
3.	AI特征分析仅对“顶尖博主库”内账号执行，每月更新一次

•	抖音、快手、小红书、视频号、B站、YouTube

每个平台详情页须包含：
3.3.2.1 平台基础信息
•	平台名称
•	核心算法逻辑（文本描述）
•	用户行为权重分布

3.3.2.2 平台榜单
榜单类型	展示数量	更新频率	包含字段
综合热度榜	TOP20	每日	排名、博主名、粉丝量、互动率
涨粉飙升榜	TOP20	每周	排名、博主名、涨粉量、涨粉率
新锐博主榜	TOP10	每周	排名、博主名、粉丝区间、代表作
3.4 领域分析子模块
3.4.1 领域分类体系
系统须按以下层级组织内容领域：
一级分类	二级分类	数据维度
短剧类	女频情感、男频爽文、AI漫剧、沙雕搞笑	市场占比、用户画像
生活类	美妆个护、母婴育儿、家装家居	市场占比、用户画像
知识类	职场教育、科技数码	市场占比、用户画像
泛娱乐	搞笑段子、抽象文化	市场占比、用户画像
3.4.2 领域详情页
每个领域详情页须包含：
•	领域概述（文本描述）
•	市场规模及占比
•	核心用户画像（年龄、性别分布）
•	该领域TOP10博主列表
3.5 顶尖博主库
3.5.1 博主基础信息
字段名	类型	说明
blogger_id	string	唯一标识
blogger_name	string	博主名称
platform	string	所属平台
category	string	所属领域
fans_count	long	当前粉丝数
fans_growth_30d	long	30天涨粉数
total_works	int	作品总数
update_freq	float	更新频率（天/条）
mcn_info	string	MCN机构（可选）
3.5.2 作品核心指标
针对博主近30天作品，计算以下指标：
指标名	类型	计算方法
avg_play_30d	long	平均播放量
max_play	long	最高播放量
avg_like_30d	long	平均点赞量
avg_comment_30d	int	平均评论量
avg_share_30d	int	平均分享量
interaction_rate	float	(点赞+评论+分享)/播放量 ×100%
hit_rate	float	播放量>100万的作品占比
avg_duration	int	平均时长（秒）
3.5.3 博主列表展示
博主列表页须支持：
•	按平台筛选
•	按领域筛选
•	按粉丝量级排序
•	按互动率排序
•	关键词搜索
4. 数据需求
4.1 数据来源
数据类型	来源	获取方式
宏观行业数据	艾媒咨询、CNNIC	手动录入+季度更新
平台榜单数据	DataEye短剧API、新榜API	每日自动抓取
博主详细数据	新榜API、飞瓜数据	每周批量拉取
动态资讯	行业媒体RSS	每日自动抓取
4.2 数据更新策略
数据表	更新策略	保留策略
宏观指标表	按更新频率覆盖	保留最新
平台榜单表	每日全量更新	保留30天历史
博主信息表	每周增量更新	永久保留
博主作品表	每月增量更新	永久保留
4.3 数据库设计
sql
-- 宏观指标表
CREATE TABLE macro_indicators (
    id INT PRIMARY KEY AUTO_INCREMENT,
    indicator_name VARCHAR(50),
    indicator_value DECIMAL(12,2),
    unit VARCHAR(20),
    update_date DATE,
    source VARCHAR(100)
);

-- 平台榜单表
CREATE TABLE platform_rankings (
    id INT PRIMARY KEY AUTO_INCREMENT,
    platform VARCHAR(20),
    ranking_type VARCHAR(20),  -- 'comprehensive'/'growth'/'new'
    rank_no INT,
    blogger_id VARCHAR(50),
    metric_value DECIMAL(12,2),
    rank_date DATE
);

-- 博主信息表
CREATE TABLE bloggers (
    blogger_id VARCHAR(50) PRIMARY KEY,
    blogger_name VARCHAR(100),
    platform VARCHAR(20),
    category VARCHAR(50),
    fans_count BIGINT,
    fans_growth_30d BIGINT,
    total_works INT,
    update_freq FLOAT,
    mcn_info VARCHAR(100),
    last_updated DATE
);

-- 博主作品指标表
CREATE TABLE blogger_metrics (
    blogger_id VARCHAR(50),
    stat_date DATE,
    avg_play_30d BIGINT,
    max_play BIGINT,
    avg_like_30d BIGINT,
    avg_comment_30d INT,
    avg_share_30d INT,
    interaction_rate FLOAT,
    hit_rate FLOAT,
    avg_duration INT,
    PRIMARY KEY (blogger_id, stat_date)
);
5. 外部接口需求
5.1 用户界面接口
接口路径	方法	说明
/api/v1/dashboard/macro	GET	获取宏观指标数据
/api/v1/platforms	GET	获取平台列表
/api/v1/platform/{platform}/rankings	GET	获取指定平台榜单
/api/v1/categories	GET	获取领域分类
/api/v1/category/{category}/top	GET	获取领域TOP博主
/api/v1/bloggers	GET	博主列表（支持分页/筛选）
/api/v1/blogger/{id}	GET	获取博主详情
5.2 第三方接口
5.2.1 DataEye短剧API
•	接口地址：https://api.dataeye.com/shortdrama/v1
•	认证方式：API Key
•	调用频率：≤100次/日
•	返回格式：JSON
5.2.2 新榜API
•	接口地址：https://open.newrank.cn/api
•	认证方式：Access Token
•	调用频率：≤1000次/日
•	返回格式：JSON
