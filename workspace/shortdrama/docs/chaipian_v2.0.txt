拆片分析功能业务需求文档 v2.0

1. 项目概述

功能名称：智能拆片分析引擎

功能定位：短剧制作自动化工具的核心AI能力模块。旨在超越基础视频信息提取，实现对视频内容、结构、镜头语言、声音、剪辑手法及市场表现潜力的多维度、深层次、自动化智能解析，为内容创作提供从技术参数到创作灵感的全面数据洞察。

核心价值：将视频分析的深度从“是什么”（基础信息）提升到“为什么好”（结构、情感、技术）以及“如何借鉴”（优化建议、市场对标），实现数据驱动的精准创作。

2. 核心业务目标

深度解构：系统化拆解爆款视频的成功要素，形成可量化的分析报告。

智能诊断：自动识别视频在内容、技术、节奏等方面的优点与潜在不足。

创作赋能：提供直接的、可操作的优化建议与创作模板，缩短学习曲线。

市场洞察：通过分析结果关联市场趋势，辅助内容决策。

3. 详细功能需求 (v2.0 升级版)

3.1. 视频基本信息分析（基础层）

需求描述：全面获取视频的物理与技术参数。

功能细则（在v1.0基础上扩充）：

核心参数：视频总时长、分辨率、帧率、文件大小、编码格式（视频/音频编解码器）。

高级参数：视频比特率、音频采样率、声道数、色彩空间。

展示形式：以清晰的参数面板展示，高亮关键信息（如是否4K、高帧率）。

3.2. 叙事与结构分析（内容层）

需求描述：自动解析视频的叙事骨架与节奏。

功能细则（在v1.0基础上深化）：

结构分段：自动识别并划分视频的叙事段落（如：黄金前3秒（钩子）、铺垫、发展、核心冲突/高潮、结尾、互动引导）。

时长占比：计算各段落的时长及占总时长的百分比，以图表形式呈现。

节奏曲线：生成视频的“情绪/节奏曲线图”，横轴为时间，纵轴为通过镜头、音乐、字幕等因素综合估算的节奏强度或情感值，直观展示起伏点。

3.3. 镜头语言分析（视觉层）

需求描述：量化分析视频的视觉表达手法。

功能细则：

景别识别：自动识别并统计特写、中景、全景、远景、大特写等镜头的数量与占比。

镜头统计：总镜头数、平均镜头时长。

切换频率：计算镜头切换速率，判断剪辑节奏是快是慢。

运动分析：识别镜头运动方式（推、拉、摇、移、跟、升降）及其使用频次。

可视化：用饼图/条形图展示景别分布，用时间点标记展示运动镜头的出现位置。

3.4. 音效与音频分析（听觉层）

需求描述：解构视频的声音设计。

功能细则（在v1.0基础上扩充）：

成分分离：识别人声对白、背景音乐、音效三类音频轨道。

对白分析：计算人声时长占比，识别是否有字幕，分析语速。

音乐分析：识别背景音乐片段，分析其情感倾向（激昂/舒缓）、出现时段。

音效统计：分类统计音效类型（如环境声、动作音、转场音效）。

质量评估：检测是否存在爆音、电流声等常见音频问题。

3.5. 内容语义分析（理解层）

需求描述：理解视频所表达的主题、人物与情感。

功能细则（新增维度）：

主题识别：通过画面与语音识别，提取视频的核心主题/关键词（如“职场反击”、“校园爱情”、“家庭温情”）。

人物识别：识别主要出镜人物数量，分析主角与配角的面部出镜时长。

场景识别：识别视频中的主要场景（如办公室、咖啡馆、街道）及其切换。

情感基调分析：综合画面、音乐、字幕，判断视频整体的情感基调（欢乐、悲伤、紧张、励志等）及其变化。

3.6. 剪辑与视觉风格分析（技术层）

需求描述：分析视频的后期制作技巧与视觉风格。

功能细则（新增维度）：

剪辑风格：判断剪辑风格倾向（如快速碎切、长镜头叙事、匹配剪辑等）。

转场分析：识别使用的转场效果（硬切、叠化、闪白、缩放等）及频率。

色彩调色：分析视频的整体色调（暖色/冷色）、色彩饱和度、对比度风格。

特效与文字：识别使用的特效（如分屏、粒子）和动态文字标题样式。

3.7. 综合优化与市场建议（应用层）

需求描述：基于以上所有分析，给出可操作的总结与建议。

功能细则（新增维度，体现核心价值）：

优势诊断报告：自动生成一段文字总结，指出该视频在结构、镜头、情绪引爆点等方面的核心优点。

改进建议：针对可优化的点（如开头不够抓人、中间段落拖沓、音效单调）提供具体建议。

同类对比：提示用户此视频与工具内“对标分析”模块中同类热门视频在结构、时长、景别上的数据差异。

市场趋势参考：根据该视频的分析标签（主题、情感、节奏），关联当前平台热门趋势，给出“此类内容近期表现”的简要提示。

4. 非功能性需求 (升级)

性能：对于5分钟以内的短视频，完整分析流程应在1-2分钟内完成，并给予进度提示。支持任务队列，用户可同时提交多个视频。

准确性：核心参数（时长、分辨率）需100%准确；AI分析维度（如景别、情感）需达到行业可用水平（如>85%准确率），并提供置信度提示。

可扩展性：采用MCP（Model Context Protocol）架构，将各分析维度（如镜头识别、音频分析、情感理解）封装为独立的工具（Tools），由后端MCP服务器提供。前端作为客户端调用，便于未来单独升级或替换某项AI能力。

结果展示：提供交互式报告面板，支持图表联动筛选（如点击节奏曲线的高峰点，定位到对应时间线的镜头列表）。提供报告导出功能（PDF/HTML）。

5. 实施路线图建议

阶段一 (基础夯实)：实现 3.1 视频基本信息​ 和 3.2 结构分析​ 的真实能力（通过FFmpeg和简单算法），并优化v1.0的界面为现代化卡片和图表。

阶段二 (核心视觉/听觉分析)：接入CV模型实现 3.3 镜头分析，接入音频处理模型实现 3.4 音效分析，形成核心竞争力。

阶段三 (AI深度理解)：接入多模态大模型，实现 3.5 内容语义分析​ 和 3.6 剪辑风格分析​ 中的高级功能。

阶段四 (智能应用)：构建数据分析后台，实现 3.7 优化与市场建议，完成从“分析工具”到“创作顾问”的升级。
