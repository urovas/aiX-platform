## 🎯 三种生成模式

### 模式一：快速生成（最简单）
**适用场景**：快速生成一批场景进行测试

**操作步骤**：
1. 选择 **"快速生成"** 标签
2. 输入生成数量（如：100）
3. 选择难度等级（简单/中等/困难/地狱）
4. 选择天气条件（可选）
5. 点击 **"开始生成"**

**示例**：生成100个中等难度的雨天场景

---

### 模式二：自然语言（最智能）
**适用场景**：用自然语言描述具体场景需求

**操作步骤**：
1. 选择 **"自然语言"** 标签
2. 在输入框中描述场景需求
3. 点击 **"智能生成"**

**支持的描述**：
- 数量："生成50个..."
- 天气："雨天"、"雾天"、"夜间"
- 车型："卡车"、"公交车"
- 遮挡："有遮挡"
- 流量："高流量"
- 行人："有行人"
- 难度："困难"、"简单"

**示例输入**：
- "生成100个雨天卡车遮挡高流量场景"
- "生成50个雾天困难场景"
- "生成20个夜间公交车有行人场景"

---

### 模式三：对抗性生成（最强大）
**适用场景**：智能发现Apollo系统的弱点

**核心逻辑**：
```
第0轮：随机生成 → 测试 → 分析失败
第1轮：聚焦高危参数空间 → 密集采样 → 测试
第2轮：进一步聚焦 → 发现细分失败模式
第3轮：精准打击 → 验证高危参数组合
```

**操作步骤**：
1. 选择 **"对抗性生成"** 标签
2. 设置初始场景数（建议100-500）
3. 选择迭代次数（建议3轮）
4. 选择执行方式（模拟/CARLA）
5. 点击 **"开始对抗"**

**预期效果**：
- 初始随机：失败率 ~20-30%
- 第1轮对抗：失败率 ~50-60%
- 第2轮对抗：失败率 ~70-80%
- 第3轮对抗：失败率可能达到90%+

## 📊 8个参数维度

| 参数 | 范围 | 说明 |
|------|------|------|
| 1. 自车速度 | 5-30 km/h | 接近路口的速度 |
| 2. 对向车速 | 30-80 km/h | 对向直行车的速度 |
| 3. 时间间隙 | 2-8秒 | 到达冲突点的时间差 |
| 4. 对向车型 | 轿车/卡车/公交 | 影响博弈行为 |
| 5. 天气 | 晴/雨/雾/夜 | 影响感知难度 |
| 6. 视野遮挡 | 有/无 | 大车遮挡小车 |
| 7. 交通流量 | 低/中/高 | 多车道车流 |
| 8. 行人/非机动车 | 有/无 | 增加复杂度 |

## 🎮 典型工作流程

### 工作流1：快速评估Apollo性能
```
1. 快速生成 → 100个中等难度场景
2. 批量测试 → 执行所有场景
3. 查看结果 → 成功率、碰撞率、超时率
4. 生成报告 → 导出测试报告
```

### 工作流2：发现Apollo弱点
```
1. 对抗性生成 → 初始500个，迭代3轮
2. 等待完成 → 查看迭代进度
3. 分析结果 → 查看高危参数组合
4. 查看案例 → 查看失败典型案例
```

### 工作流3：特定条件测试
```
1. 自然语言 → "生成50个雾天卡车场景"
2. 生成场景 → 等待生成完成
3. 查看详情 → 点击场景查看8参数
4. 导出场景 → 导出为OpenSCENARIO
```

## 💡 使用技巧

### 技巧1：从简单开始
- 先使用"快速生成"生成少量场景（20-50个）
- 熟悉系统操作后再进行大规模生成

### 技巧2：善用自然语言
- 自然语言支持组合描述
- 例如："生成30个困难雨天卡车遮挡高流量有行人场景"

### 技巧3：对抗性生成策略
- 初始场景数建议100-500
- 迭代次数建议2-3轮
- 首次使用建议选择"模拟执行"（快速）

### 技巧4：查看高危参数组合
- 对抗性生成完成后，查看"高危参数组合"列表
- 这些组合代表了Apollo的弱点
- 可用于针对性优化算法

### 技巧5：导出和分享
- 场景可以导出为OpenSCENARIO格式
- 兼容CARLA等多种仿真平台
- 可以分享给其他团队使用

## ❓ 常见问题

**Q: 生成场景需要多长时间？**
A: 快速生成100个场景约需5-10秒；对抗性生成500个×3轮约需1-2小时（模拟）或更久（CARLA）。

**Q: 如何查看生成的场景？**
A: 在Web界面的场景列表中查看，或查看 `./scenarios/unprotected_left_turn/` 目录下的JSON文件。

**Q: 对抗性生成和随机生成有什么区别？**
A: 随机生成是均匀采样参数空间；对抗性生成会分析失败案例，在高危参数空间内密集采样，更容易发现系统弱点。

**Q: 可以自定义参数范围吗？**
A: 可以，修改 `generators/unprotected_left_turn_generator.py` 中的 `parameter_ranges` 字典。

**Q: 如何集成到CI/CD流程？**
A: 使用命令行脚本，可以集成到自动化测试流程中。
